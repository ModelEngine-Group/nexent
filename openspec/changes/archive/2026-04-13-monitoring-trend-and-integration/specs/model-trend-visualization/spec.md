## 新增需求

### 需求：趋势图展示时间序列指标并带面积填充
系统 SHALL 渲染 recharts AreaChart 组件，展示 5 种可切换指标（请求数、错误率、故障率、平均耗时、成本）随时间变化的曲线，并使用线性渐变填充曲线下方区域。

#### 场景：用户查看请求数趋势
- **WHEN** 用户导航到监控页面并选择"请求数"指标
- **THEN** 系统展示 AreaChart，显示所选时间范围内的 request_count 值，使用蓝色渐变填充

#### 场景：用户切换到错误率指标
- **WHEN** 用户在指标选择器中点击"错误率"段
- **THEN** 图表更新为展示 error_rate 值，使用橙色渐变填充，Y 轴调整为百分比刻度

### 需求：趋势图支持全模型汇总视图
系统 SHALL 默认展示所有被监控模型的聚合趋势数据，将指标求和/平均后作为单条曲线展示。

#### 场景：默认聚合视图
- **WHEN** 用户查看趋势图且未选择特定模型
- **THEN** 图表展示来自 `/monitoring/trend` API 的聚合数据（request_count 求和、error_rate 求平均、cost 求和）

### 需求：趋势图支持单模型过滤
系统 SHALL 提供 Select 下拉框，允许用户将趋势数据过滤为单一指定模型。

#### 场景：用户选择单一模型
- **WHEN** 用户从模型下拉框中选择"GPT-4o"
- **THEN** 图表更新为仅展示 GPT-4o 的趋势数据（通过向 API 传递 `model_id`），下拉框显示所选模型名称

#### 场景：用户清除模型过滤
- **WHEN** 用户清除模型下拉框的选择
- **THEN** 图表恢复为展示所有模型的聚合汇总数据

### 需求：趋势图时间范围与页面工具栏联动
趋势图数据 SHALL 在页面级时间范围选择器变更时自动刷新（24h / 7d / 30d）。

#### 场景：时间范围变更触发刷新
- **WHEN** 用户在工具栏中从"最近 24 小时"切换为"最近 7 天"
- **THEN** 趋势图以 `time_range=7d` 获取新数据，并以合适的日期格式（而非小时格式）重新渲染时间轴标签
