## 1. 设计思路

### 核心原则

```
传统方式：nexent 为每个外部 KB 写 adapter（DifyAdapter、AidpAdapter...）
          → 每新增一个 KB，nexent 要改 6 处代码

本方案：  nexent 定义 Python ABC（ExternalKBAdapter），外部 KB 方继承实现适配器类
          通过 ExternalKBAdapterRegistry 注册 platform → adapter class 映射
          ExternalKnowledgeBaseService 按 DB 记录中的 platform 字段自动实例化
          → 新增外部 KB 只需：实现适配器类 + 注册到 Registry + DB 插入记录
```

> **架构演进说明**：原方案设计了容器化部署（每个适配器是独立 Docker 容器），实践中发现 in-process 方案更优——零运维开销、低延迟（无 HTTP 序列化）、共享进程资源。当前实现采用 in-process Python 抽象层。§3 的 HTTP 接口规范保留为外部适配器（HTTP 微服务）的可选契约。

### 职责划分

| 角色 | 职责 |
|------|------|
| **nexent** | 定义接口规范（ABC）、维护 Registry、管理适配器 DB 记录、统一 KB 列表/工具配置/检索编排 |
| **外部 KB 方** | 继承 `ExternalKBAdapter` ABC，实现 `platform` 属性和所需方法，注册到 `ExternalKBAdapterRegistry` |
| **用户** | 在 nexent 平台注册适配器（DB 记录）、选择知识库、配置 Agent 工具，无感知后端来源 |

### 数据流

```
用户在 nexent 配置 Agent 工具
  → GET /api/v1/kb/adapters/{id}/knowledge-bases 获取 KB 列表
  → Service 通过 Registry 实例化对应 adapter → adapter.list_knowledge_bases()
  → 用户选择 KB，配置存入 nexent DB

Agent 运行时检索
  → POST /api/v1/kb/adapters/{id}/knowledge-bases/{kb_id}/retrieve
  → Service.get_adapter(id) → adapter.search(SearchRequest)
  → 适配器翻译为外部 KB 原生检索 API（LocalKBAdapter 直接调 ES）
  → 适配器将原生结果归一化为 SearchResponse（含 V4 records[].segment[] 结构）
  → nexent 封装为 SearchResultTextMessage，推送给 Agent/前端
```
  → 适配器将原生结果归一化为标准格式返回
  → nexent 封装为 SearchResultTextMessage，推送给 Agent/前端
```

## 3. 标准接口规范（适配层必须实现）

### 3.1 通用定义

#### 3.1.1 基础 URL

```
{base_url}/api/v1
```

`base_url` 为适配层服务的根地址，例如 `http://dify-adapter:8080`。

#### 3.1.2 认证方式

```
Header: Authorization: Bearer {api_key}
```

由 nexent 统一管理和传递。`api_key` 在 nexent 注册适配器时配置，注入适配器容器环境变量后由适配层自行校验。

#### 3.1.3 统一响应格式

**成功响应**：

```json
{
  "code": 0,
  "data": { ... },
  "message": "success"
}
```

**失败响应**：

```json
{
  "code": 10001,
  "data": null,
  "message": "错误描述信息"
}
```

**分页响应（列表类接口）**：

```json
{
  "code": 0,
  "data": {
    "list": [ ... ],
    "total": 100,
    "page": 1,
    "page_size": 20,
    "has_more": false
  },
  "message": "success"
}
```

#### 3.1.4 通用字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | string | 唯一标识符（UUID 格式） |
| `name` | string | 名称 |
| `description` | string | 描述 |
| `created_at` | string | 创建时间（ISO 8601 格式，如 `2026-07-03T10:00:00Z`） |
| `updated_at` | string | 更新时间（ISO 8601 格式） |
| `status` | string | 状态标识 |

#### 3.1.5 业务状态码

| 状态码 | 说明 |
|--------|------|
| 0 | 成功 |
| 40001 | 参数错误（如必填字段缺失、格式不正确） |
| 40002 | 资源不存在（如知识库 ID 或文档 ID 无效） |
| 40003 | 资源已存在（如知识库名称重复） |
| 40004 | 资源状态不允许操作（如正在索引中的文档不允许删除） |
| 40005 | 文件格式不支持 |
| 40006 | 文件大小超出限制 |
| 50001 | 系统内部错误 |
| 50002 | 服务暂时不可用 |
| 50003 | 索引处理中，请稍后重试 |

#### 3.1.6 HTTP 状态码映射

| HTTP Code | 场景 |
|-----------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 认证失败（API Key 无效或已过期） |
| 403 | 无权访问该资源 |
| 404 | 资源不存在 |
| 409 | 资源冲突（如名称重复） |
| 422 | 请求语义错误 |
| 429 | 请求过于频繁，触发限流 |
| 500 | 服务器内部错误 |
| 503 | 服务暂时不可用 |

### 3.2 适配层扩展接口（非 V4 标准，自定义）

适配器除实现 V4 标准接口外，还需要实现以下扩展接口用于 nexent 容器生命周期管理和能力探测。这部分接口不走 `/api/v1` 前缀。

#### 3.2.1 健康检查 `GET /health`

nexent 用于检查适配层及外部 KB 连接是否正常（在容器编排、监控场景使用）。

**响应**：

```json
{
  "status": "ok",
  "platform": "dify",
  "version": "1.0.0",
  "external_kb_reachable": true
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | string | `"ok"` 或 `"error"` |
| `platform` | string | 平台标识，用于 nexent 侧展示 |
| `version` | string | 适配层版本号 |
| `external_kb_reachable` | bool | 外部 KB 是否可达 |

#### 3.2.2 能力声明 `GET /capabilities`

声明本适配层支持哪些 V4 标准接口。nexent 据此控制 UI 可见性与调用降级。

**响应**：

```json
{
  "create_knowledge_base": false,
  "delete_knowledge_base": false,
  "update_knowledge_base": false,
  "upload_document": false,
  "delete_document": false,
  "list_documents": false,
  "query_document_status": false,
  "download_document": false,
  "list_models": false,
  "search_modes": ["semantic_search", "keyword_search", "hybrid_search"],
  "supports_rerank": true,
  "supports_multimodal": true
}
```

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `create_knowledge_base` | bool | false | 是否支持创建知识库 |
| `delete_knowledge_base` | bool | false | 是否支持删除 |
| `update_knowledge_base` | bool | false | 是否支持更新元信息 |
| `upload_document` | bool | false | 是否支持上传文档 |
| `delete_document` | bool | false | 是否支持删除文档 |
| `list_documents` | bool | false | 是否支持列文档 |
| `query_document_status` | bool | false | 是否支持查询文档处理状态 |
| `download_document` | bool | false | 是否支持获取文件下载 URL |
| `list_models` | bool | false | 是否支持获取模型列表 |
| `search_modes` | string[] | `["semantic_search"]` | 支持的检索方式：`semantic_search` / `keyword_search` / `hybrid_search` |
| `supports_rerank` | bool | false | 适配层/外部KB是否原生 rerank |
| `supports_multimodal` | bool | false | 是否支持多模态（图片/表格） |

> 注：`POST /retrieve`（V4 标准中的跨知识库检索接口）对所有适配层都是 **必须实现** 的，不通过 capabilities 声明。

### 3.3 知识库管理（V4 标准）

> 本节所有接口路径均相对于 `{base_url}/api/v1`。

#### 3.3.1 创建知识库 `POST /knowledge-bases`

创建一个空的知识库。

**入参（Request Body）**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `name` | string | 是 | 知识库名称，同一租户下唯一。长度限制 1-100 字符。 |
| `description` | string | 否 | 知识库描述。长度限制 0-500 字符，不传默认为空字符串。 |
| `embedding_model` | string | 否 | 向量嵌入模型名称，如 `text-embedding-3-small`。不传则使用平台默认模型。 |
| `is_multimodal` | boolean | 否 | 是否启用多模态支持。`true` 表示启用，`false` 表示不启用（默认）。启用后，知识库支持处理图片、图表等视觉内容。 |
| `vision_model` | string | 否 | 视觉模型名称，如 `gpt-4o-vision`。仅在 `is_multimodal` 为 `true` 时必填，用于解析和向量化图片内容。 |

**请求示例**：

```json
{
  "name": "产品文档库",
  "description": "存放产品相关的技术文档",
  "embedding_model": "text-embedding-3-small",
  "is_multimodal": false,
  "vision_model": null
}
```

**出参（Response Data）**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `knowledge_base_id` | string | 知识库唯一 ID（UUID 格式）。 |
| `name` | string | 知识库名称。 |
| `description` | string | 知识库描述。 |
| `embedding_model` | string | 使用的嵌入模型名称。 |
| `document_count` | integer | 当前知识库中文档数量，新建时为 0。 |
| `status` | string | 知识库状态，固定返回 `active`。 |
| `created_at` | string | 创建时间（ISO 8601 格式）。 |
| `updated_at` | string | 更新时间（ISO 8601 格式），新建时与 `created_at` 相同。 |

**响应示例**：

```json
{
  "code": 0,
  "data": {
    "knowledge_base_id": "c42e2a6e-40b3-4330-96f8-f1e4d768e8c9",
    "name": "产品文档库",
    "description": "存放产品相关的技术文档",
    "embedding_model": "text-embedding-3-small",
    "document_count": 0,
    "status": "active",
    "created_at": "2026-07-03T10:00:00Z",
    "updated_at": "2026-07-03T10:00:00Z"
  },
  "message": "success"
}
```

#### 3.3.2 获取知识库列表 `GET /knowledge-bases`

**入参（Query Parameters）**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `page` | integer | 否 | 页码，从 1 开始计数。不传默认值为 1。 |
| `page_size` | integer | 否 | 每页返回数量。不传默认值为 20，最大允许值为 100。 |

**响应 Data 结构**（V4 标准 `list/total/page/page_size/has_more`）：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `list` | array | 知识库对象列表。 |
| `list[].id` | string | 知识库唯一 ID（UUID 格式）。 |
| `list[].name` | string | 知识库名称。 |
| `list[].description` | string | 知识库描述。 |
| `list[].document_count` | integer | 文档数量。 |
| `list[].status` | string | 知识库状态：`active` / `inactive`。 |
| `list[].created_at` | string | 创建时间（ISO 8601 格式）。 |
| `list[].updated_at` | string | 更新时间（ISO 8601 格式）。 |
| `total` | integer | 知识库总数。 |
| `page` | integer | 当前页码。 |
| `page_size` | integer | 每页数量。 |
| `has_more` | boolean | 是否还有更多数据。 |

**响应示例**：

### 3.4 模型管理（V4 标准，按能力声明可选）

| `page` | integer | 当前页码。 |
| `page_size` | integer | 每页数量。 |
| `has_more` | boolean | 是否还有更多数据。 |

**响应示例**：

```json
{
  "code": 0,
  "data": {

### 3.6 知识检索（V4 标准，**必须实现**）

#### 3.6.1 跨知识库检索 `POST /retrieve`

在多个指定的知识库中同时进行语义/关键词检索，返回相关性最高的文档片段。**这是 Adapter 必须实现的检索接口**。

**入参（Request Body）**：

| 字段名 | 类型 | 必填 | 说明 |

## 4. 适配器开发规范（in-process）

> **原 Docker 镜像打包规范已废弃**。当前架构采用 in-process Python 抽象层，适配器作为 nexent 进程内的 Python 类运行，无需独立容器。

### 4.1 实现步骤

1. 在 `sdk/nexent/core/knowledge_base/` 下新建文件（如 `dify_adapter.py`），继承 `ExternalKBAdapter` ABC
2. 实现必须方法：`platform` 属性、`health_check()`、`get_capabilities()`、`list_knowledge_bases()`、`get_knowledge_base()`、`search()`
3. 按需实现可选方法（未实现则抛 `NotImplementedError`）：`create/update/delete_knowledge_base()`、`list/upload/delete_document()`、`get_document_status()`、`get_document_download_url()`
4. 在文件末尾注册：`ExternalKBAdapterRegistry.register("dify", DifyKBAdapter)`
5. 确保模块在启动时被导入，使 `register()` 被调用
6. 通过 `POST /api/v1/kb/adapters` 插入 DB 记录，`platform` 字段与注册的字符串一致

### 4.2 配置传递

通过 DB 的 `external_kb_config` JSON 列传递：

```json
{
  "url": "https://api.dify.ai/v1",
  "api_key": "app-xxx",
  "extra": { "tenant_id": "optional-extra-config" }
}
```

`ExternalKBAdapter.__init__()` 将此解析为 `self._url`、`self._api_key`、`self._extra`。

### 4.3 健康检查约定

`health_check()` 必须返回：

```python
{
    "status": "ok" | "error",
    "platform": "dify",
    "version": "1.0.0",
    "external_kb_reachable": True | False,
}
```

结果会持久化到 DB 的 `health_status` 和 `last_health_check` 字段。

### 5.1 适配器注册表

数据库表 `external_kb_adapter_t`，存储已注册的适配器信息。

**`backend/database/db_models.py`** ORM 定义（当前实现）：

```python
class ExternalKBAdapter(TableBase):
    """
    Registered external KB adapter instances.
    For in-process adapters (e.g. LocalKBAdapter), service_host and
    container fields are not used.
    """
    __tablename__ = "external_kb_adapter_t"
    __table_args__ = {"schema": SCHEMA}

    adapter_id = Column(Integer, Sequence("external_kb_adapter_t_adapter_id_seq", schema=SCHEMA),
                        primary_key=True, nullable=False,
                        doc="Adapter ID, auto-increment primary key")

**`backend/services/external_kb_service.py`**：

```python
from nexent.core.knowledge_base.platform_adapters import (
    ExternalKBAdapter, ExternalKBAdapterRegistry, SearchRequest,
)


def _build_adapter(adapter_record: Dict[str, Any], user_id: str = "") -> ExternalKBAdapter:
    """Build a live adapter instance from a DB record dict."""
    platform = adapter_record.get("platform", "").lower()
    config = adapter_record.get("external_kb_config") or {}
    if isinstance(config, dict):
        config = {
            **config,
            "tenant_id": adapter_record.get("tenant_id", ""),
            "user_id": user_id or adapter_record.get("user_id", ""),
        }
    return ExternalKBAdapterRegistry.instantiate(platform, config)


class ExternalKnowledgeBaseService:
    """Orchestration layer. All methods are static; no per-request state."""

    @staticmethod
    def get_adapter(adapter_id: int, tenant_id: str) -> ExternalKBAdapter:
        from database.external_kb_adapter_db import get_adapter_by_id
        adapter = get_adapter_by_id(adapter_id, tenant_id)
        if not adapter:
            raise ValueError(f"Adapter {adapter_id} not found for tenant {tenant_id}")
        if not adapter.get("enabled"):
            raise ValueError(f"Adapter {adapter_id} is disabled")
        return _build_adapter(adapter)

    @staticmethod
    def list_all_external_knowledge_bases(tenant_id: str, keyword: Optional[str] = None) -> List[Dict[str, Any]]:
        from database.external_kb_adapter_db import query_adapters_by_tenant
        adapters = query_adapters_by_tenant(tenant_id, enabled_only=True)
        all_external_kbs = []
        for adapter in adapters:
            try:
                adapter_obj = _build_adapter(adapter)
                kbs = adapter_obj.list_knowledge_bases(keyword=keyword, page_size=200)
                for kb in kbs:
                    kb_dict = kb.to_dict()
                    kb_dict.update({
                        "adapter_id": adapter["adapter_id"],
                        "adapter_name": adapter["name"],
                        "platform": adapter["platform"],
                        "source": "external",
                    })
                    all_external_kbs.append(kb_dict)
                adapter_obj.close()
            except Exception as exc:
                logger.warning("Failed to list KBs from adapter %s: %s", adapter.get("name"), exc)
        return all_external_kbs

    @staticmethod
    def retrieve(adapter_id: int, tenant_id: str, request) -> dict:
        adapter = ExternalKnowledgeBaseService.get_adapter(adapter_id, tenant_id)
        if isinstance(request, dict):
            kb_ids = request.get("kb_ids") or request.get("knowledge_base_ids", [])
            retrieval_model = request.get("retrieval_model") or {}
            req = SearchRequest(
                query=request["query"],
                kb_ids=kb_ids,
                top_k=retrieval_model.get("top_k", request.get("top_k", 5)),
                search_mode=retrieval_model.get("search_method", request.get("search_mode", "hybrid")),
                score_threshold=retrieval_model.get("score_threshold", request.get("score_threshold", 0.0)),
                rerank=retrieval_model.get("reranking_enable", request.get("rerank", False)),
            )
        else:
            req = request
        try:
            response = adapter.search(req)
            return response.to_dict()
        finally:
            adapter.close()
```

### 7.2 工具配置：新增 ExternalKnowledgeSearchTool

```python
import json
from smolagents import Tool
from sdk.nexent.core.knowledge_base.external_kb_client import ExternalKBClient, SearchRequest
from sdk.nexent.core.utils.tools_common_message import SearchResultTextMessage
from sdk.nexent.core.utils.constants import RERANK_OVERSEARCH_MULTIPLIER


class ExternalKnowledgeSearchTool(Tool):
    """
    Search external knowledge bases via registered adapters.
    Users only see nexent platform; the adapter handles translation.
    """
    name = "external_kb_search"
    description = "Performs a knowledge base search on external platforms..."
    description_zh = "执行外部知识库检索并返回最相关的结果。"

    inputs = {
        "query": {"type": "string", "description": "The search query to perform."},
    }
    output_type = "string"
    category = ToolCategory.SEARCH.value
    tool_sign = ToolSign.EXTERNAL_KB_SEARCH.value  # 新增枚举值

    def __init__(self,
                 adapter_id: int = Field(...),
                 kb_ids: str = Field(..., description="JSON string array of KB IDs"),
                 kb_display_names: str = Field(default="", description="JSON string array of KB names for prompt"),
                 top_k: int = Field(default=5),
                 search_mode: str = Field(default="hybrid"),
                 score_threshold: float = Field(default=0.0),
                 rerank: bool = Field(default=False),
                 observer: MessageObserver = Field(default=None, exclude=True),
                 client: ExternalKBClient = Field(default=None, exclude=True),
                 embedding_model_config: EmbeddingModelConfig = Field(default=None, exclude=True),  # per-request model config passthrough
    ):
        self.adapter_id = adapter_id
        self.kb_ids = json.loads(kb_ids) if isinstance(kb_ids, str) else kb_ids
        self.kb_display_names = json.loads(kb_display_names) if isinstance(kb_display_names, str) else (kb_display_names or [])
        self.top_k = top_k
        self.search_mode = search_mode
        self.score_threshold = score_threshold
        self.rerank = rerank
        self.observer = observer
        self.client = client
        self.embedding_model_config = embedding_model_config  # None if adapter doesn't require it

    def forward(self, query: str) -> str:
        if not self.client:
            raise RuntimeError("ExternalKBClient not injected")

        effective_top_k = self.top_k
        if self.rerank:
            effective_top_k = effective_top_k * RERANK_OVERSEARCH_MULTIPLIER

        request = SearchRequest(
            query=query,
            kb_ids=self.kb_ids,
            top_k=effective_top_k,
            search_mode=self.search_mode,
            score_threshold=self.score_threshold,
            rerank=self.rerank,
            embedding_model=self.embedding_model_config,  # per-request passthrough
        )

        response = self.client.search(request)

        # 封装为 SearchResultTextMessage（复用现有契约）
        search_results_return = []
        for idx, result in enumerate(response.results):
            msg = SearchResultTextMessage(
                title=result.title or result.filename,
                text=result.content,
                url=result.url,
                filename=result.filename,
                score=result.score,
                score_details=result.score_details or {},
                cite_index=self.record_ops + idx,
                search_type=self.name,
                tool_sign=self.tool_sign,
                source_type="external",
            )
            search_results_return.append(msg.to_model_dict())

        # 推送完整结果给 observer（前端渲染）
        if self.observer:
            self.observer.add_message("", ProcessType.SEARCH_CONTENT,
                                      [msg.to_dict() for msg in search_results])

        return json.dumps(search_results_return, ensure_ascii=False)
```

class SearchReq(BaseModel):
    query: str
    kb_ids: list
    top_k: int = 5
    search_mode: str = "hybrid"
    score_threshold: float = 0.0
    rerank: bool = False
    filters: dict = {}


@app.post("/search")
def search(req: SearchReq, authorization: str = Header(None)):
    verify_auth(authorization)
    # 多 KB 联合检索
    all_results = []
    for dataset_id in req.kb_ids:
        resp = httpx.post(
            f"{DIFY_URL}/datasets/{dataset_id}/retrieve",
            headers={"Authorization": f"Bearer {DIFY_KEY}"},
            json={"query": req.query,
                  "retrieval_model": {"search_method": req.search_mode,
                                      "top_k": req.top_k}},
        )
        for record in resp.json().get("records", []):
            segment = record.get("segment", {})
            all_results.append({
                "content": segment.get("content", ""),
                "title": segment.get("document", {}).get("name", ""),
                "url": "",
                "filename": segment.get("document", {}).get("name", ""),

                "score_details": {},
                "kb_id": dataset_id,
                "document_id": segment.get("document", {}).get("id", ""),
                "metadata": segment,
            })
    # 按 score 排序，取 top_k
    all_results.sort(key=lambda x: x["score"], reverse=True)
    all_results = all_results[:req.top_k]
    return {"results": all_results, "total": len(all_results), "query_time_ms": 0}

### 10.1 适配器注册（in-process）

```
1. 开发者实现适配器类（继承 ExternalKBAdapter），注册到 Registry：
   ExternalKBAdapterRegistry.register("dify", DifyKBAdapter)

   {

     "external_kb_config": {
       "url": "https://api.dify.ai/v1",

     },
     "enabled": true
   }
   → ExternalKnowledgeBaseService.register_adapter()
   → external_kb_adapter_db.create_adapter()

3. 可选：POST /api/v1/kb/adapters/{id}/health 验证连通性
   → Service.check_health() → adapter.health_check() → health_status 持久化到 DB

   → Service.refresh_capabilities() → adapter.get_capabilities() → capabilities 持久化到 DB

```
